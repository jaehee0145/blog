Git - flow 란 무엇인가? 

1. 브랜치 관리 전략 (branch management strategy) 중의 하나로
2. 메인 브랜치(master, develop) 와 일정 기간만 사용하는 보조 브랜치(feature, realese, hotfix) 를 이용하는 방법이다.
3. 브랜치마다 역할, in/out이 정해져 있어 복잡하다. 
4. 명령어 set을 제공해서 쉽게 접근 가능하다.
